# 全平台内容矩阵系统

## 项目概述

这是一个系统化内容创作工作空间，覆盖公众号（4账号矩阵）、小红书、抖音、视频号，同时支持客户交付（美容行业获客）。

核心理念：
- 记忆系统：AI 知道以前写过什么，避免重复造轮子
- 素材复用：框架、表达、案例反复使用
- 方法论沉淀：每次创作给系统添砖加瓦
- 数据驱动：凭数据迭代，不凭感觉

## 工作流

### 从想法到发布的5步流程

1. 记录选题：碎片想法 -> `01-选题管理/00-选题记录.md`
2. 深化选题：检索素材库 -> 建议复用 -> 生成文稿 -> `01-选题管理/01-待深化/`
3. 分配到平台生产：`01-选题管理/02-待生产/` -> 各平台生产线 -> `02-内容生产/{平台}/生成内容/`
4. 标题/封面/开头优化
5. 发布 -> `01-选题管理/03-已发布/` -> 数据复盘 -> 反哺方法论

### 一题多吃流程

同一个选题可以适配多个平台：
- 公众号：长文（1500-3000字），4账号差异化
- 小红书：信息图6页 或 情绪冲突文字帖
- 抖音/视频号：短视频脚本（钩子+正文+CTA）

## 素材库检索规则

生产内容前，AI 必须按以下顺序检索素材库：
1. `03-素材库/核心概念库/` - 有没有相关理论框架？
2. `03-素材库/金句库/` - 搜索带 `#选题` 标签的条目（从爆款评论区筛选的选题种子，优先匹配）；带 `#框架` 的条目可作为内容骨架；仅当 Brief 开启时才引用金句原文
3. `01-选题管理/选题规划/金句选题池.md` - 优先命中已沉淀的高潜选题种子（状态含待筛选/待深化）
4. `01-选题管理/03-已发布/` - 有没有相关已发布内容？可复用？
5. `03-素材库/爆款文稿库/` - 有没有可参考的爆款结构？
6. `02-内容生产/{当前平台}/resources/` - 平台专属素材

找到相关内容时，建议复用而非重新创作。

## 硬规则

### 内容生产
- 四账号标题公式互斥，前4字不重复
- 流程直达，不反复确认（禁止"是否开始/是否确认执行"类废确认）
- 金句库仅在 Brief 明确 `是否调用金句库：是` 时调用，且只能用已加载的金句
- 富贵模块加载后必须先产出千机塔画像，正文体现事实/感受/想象至少两层
- 小红书双模必须二选一（信息图6页 / 情绪冲突文字帖），禁止混合
- 情绪冲突模式正文禁止CTA，承接全部放置顶评论

### 合规与风控
- 不写"包过/保收益/保证疗效"
- 不做人身攻击、不煽动群体对立
- 不凭空编造"名言"或虚构数据
- 数据表达尽量给出处或标注"样本观察"
- 有争议观点必须给反驳安全阀

### 文件命名规范
- 选题：`{YYYY-MM-DD}-{短标题}.md`
- 公众号生成内容：`{prefix}-{YYYYMMDD}-{短标题}.md`（prefix: gongchang/ipgc/zengzhang/shizhan）
- 小红书/抖音/视频号：`{账号}-{YYYYMMDD}-{短标题}.md`

### 交付合同格式
所有平台的最终输出必须使用统一标记：
```
<!--FILES_JSON_START-->
{"date":"YYYY-MM-DD","files":[{"path":"..."},{"path":"..."}]}
<!--FILES_JSON_END-->
<!--FILE_START-->
{path}
{content}
<!--FILE_END-->
```
