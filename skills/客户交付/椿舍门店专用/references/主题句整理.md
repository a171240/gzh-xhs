# 主题句整理

这份整理只收 **金句库 / 对标链接内容库里真实存在、可追溯来源** 的句子，供椿舍深口播调用。

## 现在的三层分工

### 1. `topic_seed`

- 只指带 `#选题` 的爆款评论/金句。
- 可以直接作为一级题源。
- 负责：
  - 生成选题
  - 给标题定角度
  - 给开头提供句感
- 命中后必须遵守：
  - 至少 1 个标题保留原评论的核心句感
  - 不能机械照抄原话
  - 正文开头可以承接句感，但必须翻成椿舍老板会说的话

### 2. `theme_only`

- 默认先使用 `assets/高优先主题句库.json`
- `assets/主题句库.json` 作为大池子回退和轮换
- **高优先主题句库里的句子默认全部是 `theme_only`**
- 负责：
  - 定立场
  - 定情绪
  - 定结尾方向
- 不负责：
  - 直接当正文主句
  - 直接当老板台词

### 3. `body_ready`

- 正文真正可播的句子，优先来自：
  - 差评原话
  - 老板口语库
  - 顾客翻译句库
- 这部分才是深口播正文的默认主料。

## 当前硬规则

- `topic_seed` 优先级高于关键词和椿舍本地选题池。
- `theme_only` 只能定方向，不能原样直接塞进正文。
- 后续如果有句子真的像老板会说的话，再单独人工升成 `body_ready`。
- 同一条文案里，显性金句感表达最多出现 `1` 次；一旦生硬，直接降级为 `theme_only`。

## 为什么这样改

前一版的问题已经很明确：

- 真实主题句是真的
- 但很多句子不适合直接播
- 一旦原样进正文，就会变成：
  - 方法论老师口气
  - 抽象鸡汤口气
  - 文艺总结口气

所以现在统一收口：

- **`topic_seed` 负责“起题”**
- **`theme_only` 负责“定方向”**
- **`body_ready` 负责“真正开口说”**

## 对标链接调用边界

- 对标链接只允许贡献：
  - 开头方式
  - 推进节奏
  - 信任建立逻辑
  - 转折结构
- 对标链接原始正文，不直接进入标题和正文。
- 只有当某句已经沉淀进金句库并带 `#选题` 时，才按 `topic_seed` 规则处理。

## 这些句子保留真实来源，但默认不升 `topic_seed`

以下这类句子，仍然保留在资产里做主题参考，但默认不允许直接升成起题种子：

- `对于其他人，只做筛选，不去教育。`
- `做好当下该做的事情，就已经非常非常好了`
- `改变的初期总是不太舒服的`
- `所以说你知道他不会掠夺你...`
- `所有的细节都是为了专属感...`
- `一个粗糙的开始，就是最好的开始`
- `通过服务给顾客带来满意效果，获得顾客的尊重...`
- `你能感觉到他不图你任何的东西...`

原因很简单：

- 它们能定主题
- 但一旦拿来起题或直接播出去，就会显得假、飘、悬、像老师上课

## 调用建议

- `信任怀疑`
  - `topic_seed` 负责定“边界感 / 不扫兴 / 信任”
  - 差评原话负责写“推销、变脸、敷衍收尾”
  - 老板口语负责把人接住
- `问题修复`
  - `topic_seed` 或 `theme_only` 负责定“先稳住、别猛做”
  - 差评原话负责写“没清干净、时间不够、做完更糟”
  - 老板口语负责给“先做基础”的低压力承接
- `放松养护`
  - `topic_seed` 或 `theme_only` 负责定“别再给人添负担”
  - 差评原话负责写“根本放松不了、全程还在推销”
  - 老板口语负责给“先把人放松下来”的承接
- `本地找店`
  - `topic_seed` 或 `theme_only` 负责定“值不值得试一次”
  - 差评原话负责写“约不上、态度差、怕白跑”
  - 老板口语负责给“第一次先做基础项目”的承接
