# 椿舍审稿机制

这份规则只做一件事：
**审“顺不顺、像不像人话、前后有没有接上”。**

它不是格式检查，也不是字数检查。
只要读起来像说明书、像门店制度、像拼接稿，就视为没过。

## 审稿目标

合格稿必须同时满足：
- 上下文顺着走，不跳主语，不断线
- 口语化，像老板真的会这样说
- 有生活感，不是流程感
- 金句自然嵌进去，不像贴标签
- 规矩能托底，但不抢戏

## 核心原则

- 去 AI 味，不等于删掉连贯性
- 去套路，不等于删掉细节
- 自然表达，不等于写成碎片堆
- 优先保留“动作、场景、时间、后果”，少保留抽象解释
- 口语化不是加语气词，而是像人会这么说

## 一票否决项

出现任一项，直接重写，不做小修：
- 开头在讲她，第二段突然变成讲“我们店”
- 前面在写放松，后面突然切成项目说明书
- 句子本身没错，但前后接不上，像硬拼起来的
- 一段里连续出现多个“但 / 其实 / 所以 / 而是 / 只是”
- 一句话塞两个以上判断，读完不知道重点在哪
- 金句一拿掉，正文就塌了
- 规矩写得像门店须知，不像安抚
- 全文都在解释，没有一句像真人会说的话

## 朗读测试

起草完成后，必须按口播方式在脑内或出声读一遍。

默认标准：
- 单句尽量 `12-24` 字
- 最长句尽量不超过 `28-32` 字
- 一句只说一个意思
- 连续长句不能超过 2 句

只要出现下面任一情况，就必须改：
- 一口气读不顺
- 读到一半想回看前一句
- 需要自己帮它补逻辑
- 句子像写出来的，不像说出来的

## 连贯性检查

每篇必须只有一条主线：

`这个人为什么会在这个时候搜这个词 -> 她最怕什么 -> 她真正想要什么 -> 椿舍怎么接住她`

逐段检查：
- 开头是否落在一个具体时刻，而不是泛泛判断
- 中段是否仍然围着同一个冲突瞬间，没有跑题
- 结尾是否回应了前面的担心，而不是另起一套价值观

## 口语化检查

优先保留这些感觉：
- 短句
- 碎句
- 留白
- 评论区会出现的人话

优先删掉这些味道：
- 书面汇报感
- 运营腔
- FAQ 腔
- 讲义腔
- 过度完整的逻辑连接词

替换原则：
- 不说“用户核心诉求”，说“她真正怕的是”
- 不说“提供情绪价值”，说“别让人更累”
- 不说“建立信任关系”，说“别让她防着你”

### A级禁用句式

这些句式默认直接判死，不做保留：
- `这不是X，是Y`
- `你要的不是X，是Y`
- `重要的不是X，而是Y`
- `真正的X不是Y，是Z`
- 任何完整的“不是……是……”鸡汤对比句
- `更扎心的是`
- `更要命的是`
- `最尴尬的是`
- `你知道吗`
- `你猜怎么着`
- `换句话说`
- `也就是说`
- `由此可见`
- `这意味着`
- `总的来说`
- `说到底`
- `说起这个`
- `最近有个谁问我`

### B级谨慎连接词

这些不是不能用，但一段里不要连续堆：
- `但是 / 可是 / 不过`
- `因为 / 所以`
- `然后 / 接着`
- `原来 / 结果`

判断原则：
- 删掉之后如果会断，就留
- 能用动作或细节顶上，就删
- 一段里连续别超过 2 个

### 更自然的过桥方式

优先这样接，而不是硬写逻辑词：
- 用动作接：`她把手机扣在桌上`
- 用细节接：`雨越下越大`
- 用场景接：`到店门口的时候`
- 用时间接：`第二天一早`
- 用对话接：`我说先不要，她点了点头`

## 金句检查

金句只能借力，不能扛主梁。

检查顺序：
1. 去掉金句，正文还能不能成立
2. 金句放进去后，语气有没有变假
3. 金句是不是在替正文说话，而不是抢正文的话

只要金句导致绕口、拔高、突兀，直接删，不硬留。

## 规矩检查

规矩的作用是托底，不是宣誓。

正确用法：
- 写成顾客能感知到的安全感
- 写成“她不会被怎样对待”
- 写成“来了以后不用再多防一层”

错误用法：
- 列制度
- 背原则
- 讲我们店多专业

## 最终交付规则

默认先做两轮：
1. 起草首稿
2. 按这份机制复审并重写

复审通过前，不得交付。

默认只交付净稿。
除非用户明确要看调试信息，否则不要把“QC 过程”写进成品里。

## 三个灵魂拷问

交付前必须过这 3 关：
1. 朗读测试：一口气能不能读顺
2. 理解测试：不回看上句，能不能马上懂
3. 真实测试：像不像老板真的会这么说

只要有 1 条不过，就继续改。
