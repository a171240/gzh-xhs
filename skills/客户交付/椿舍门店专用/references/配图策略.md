# 椿舍配图策略

> 这份文档只定义椿舍 `配图=是` 时的单张封面规则。
> 参考来源：
> - `skills/自有矩阵/小红书图片生成.md`
> - `02-内容生产/小红书/templates/视觉风格库.md`
> - `04-数据与方法论/方法论沉淀/封面方法论.md`
>
> 吸收的是“小红书单张封面”的通用规则，不引入 6 页信息图、多图轮播、CTA 页和图片 manifest 结构。

---

## 1. 使用边界

- 只服务椿舍口播稿的封面 sidecar。
- 只做单张小红书封面，不做 P2-P6，不做正文配图，不做图片 API 调用。
- 封面文案直接复用成稿主标题，不额外再造一套封面标题。
- 不放门店信息、价格、优惠、预约、CTA。

## 2. 从知识库吸收什么

- 比例前置：提示词第一句必须是 `画幅比例3:4竖版。`
- 小红书结构：单张封面、手机端高可读、留白充足、标题必须一眼读完。
- 字体写法：使用专业中文字体描述，不写“字体好看”“ins 风字体”这类空词。
- 画面写法：描述场景、材质和版式，不堆砌关键词。
- 负面提示词：必须显式给出，避免乱码、水印、英文字母、文字扭曲、复杂背景。
- 封面文案方法：优先使用冲突式、提问式、记录感标题，而不是信息图式模块堆砌。

## 3. 不吸收什么

- 不吸收信息图 `P1-P6` 六页结构。
- 不吸收 `body-01..03` 多图轮播结构。
- 不吸收咨询报告感、高密度数据可视化排版。
- 不吸收固定 CTA 页、引流页、群聊页。
- 不吸收实拍人物、产品展示、门店环境图作为默认封面方向。

## 4. 统一封面骨架

所有模板都必须遵守这一骨架：

1. `画幅比例3:4竖版。`
2. `【图片类型】小红书封面`
3. `【封面文案】` 直接放最终主标题
4. `【版式】` 说明标题如何摆放
5. `【视觉风格】` 说明背景、留白、整体氛围
6. `【中文字体描述】` 说明字重、风格、排版
7. `【标题高亮】` 说明是否高亮单词
8. `【画面元素】` 说明允许的场景/材质元素
9. `【负面提示词】` 单独给出

## 5. 模板体系

### A 暖调文字海报

- 适用：普通观点、问题修复、地名类标题默认落 A。
- 版式：三行冲突式或单句大字式，标题居中偏上。
- 视觉：暖米白到浅杏色渐变，轻纸质肌理，留白 40-50%。
- 字体：圆润的现代无衬线黑体，加粗，整句优先可读。

### B 手写感便签

- 适用：含“记到现在 / 说了句话 / 三个字 / 说了句实话 / 笑了一下”的记录感标题。
- 版式：像随手写下的一句真心话，字面轻微倾斜。
- 视觉：奶油色便签纸、纸张阴影、暖光晕染。
- 字体：手写体或行楷风格，但字必须清晰。

### C 对话气泡

- 适用：含“她说 / 跟我说 / 问我 / 跟我聊 / 发消息”的对话触发类标题。
- 版式：单个主气泡承接标题，不做消息列表。
- 视觉：浅米色背景 + 白色圆角气泡 + 柔和阴影。
- 字体：圆润黑体，加粗，像聊天截图里的重点句。

## 6. 模板优先级

- 固定优先级：`B 手写感 > C 对话气泡 > A 默认`
- 例子：`做完脸她说了句话我记到现在` 必须命中 B，不允许因为含“她说”误落 C。

## 7. 高亮词规则

- 目标：只高亮一个真正影响点击的词，不切碎整句标题。
- 按固定优先级找首个命中词：
  - `动作`
  - `变了`
  - `不用换了`
  - `不用回消息`
  - `睡着了`
  - `推销`
  - `委屈`
  - `心累`
  - `难受`
  - `犹豫`
  - `舒服`
  - `踏实`
  - `轻了`
  - `笑了`
  - `做坏了`
  - `爆痘`
  - `红了`
  - `疼`
- 没命中就写 `无`，不要为了高亮而硬造词。

## 8. 负面提示词

通用负面提示词：

`人物照片、产品图、英文字母、二维码、水印、冷色调、科技感、3D效果、卡通风格、文字变形扭曲、文字模糊、乱码、复杂背景、人物变形`

## 9. 椿舍调性提醒

- 要像小红书封面，不像公众号头图。
- 要有情绪停顿感，不要信息图报告感。
- 要让人觉得“这句跟我有关”，不是“这页信息很多”。
- 封面负责让人停下来，正文负责解释，不要让封面替正文说完全部内容。
