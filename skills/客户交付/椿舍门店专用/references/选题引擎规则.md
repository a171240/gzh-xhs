# 椿舍选题引擎规则

> 这份文件只服务 `chunshe_wj` 的选题阶段。  
> 作用不是教模型自由发散，而是把椿舍的题从“固定 7 题”升级成“关键词进入后，可稳定扩成多个可发角度”。

## 1. 选题不是随机发散

椿舍的选题必须同时满足四件事：

1. 有入口
   - 这个题得能挂在真实搜索词上。
2. 有人感
   - 这个题得落到一个具体的不舒服时刻。
3. 有承接
   - 这个题最终得能被椿舍的真实规矩接住。
4. 有新鲜度
   - 这个题不能只是上次那篇换几个词。

## 2. 四类题源

### 2.1 关键词题源
- 问题修复词
- 信任/怀疑词
- 放松养护词
- 本地找店词

### 2.2 差评题源
- 被推销
- 被加项
- 被吓唬
- 时长缩水
- 手法敷衍
- 环境不舒服
- 预约麻烦
- 态度冒犯

### 2.3 金句题源
- 只用于放大表达，不主导题目来源。
- 只允许为题目提供：
  - 情绪命名
  - 关系判断
  - 收口方式

### 2.4 门店规矩题源
- 说不要，话题就停
- 不强推
- 不乱加项
- 按次可做
- 不适合直接说
- 流程、时长、合理预期说清楚

## 3. 六类题角

### 3.1 痛点确认
- 把“她最不舒服的那一秒”说出来。

### 3.2 防御拆解
- 把“她为什么一直不敢做”说清楚。

### 3.3 规矩托底
- 不是先说项目，而是先说这家店怎么不消耗人。

### 3.4 向往画面
- 不是夸张变美，而是“这次终于没那么累”。

### 3.5 本地决策
- 不是泛泛推荐，而是给她一个不白跑的筛店顺序。

### 3.6 关系/身份翻译
- 不是项目解释，而是把她真正买的状态、体面、被照顾感翻译出来。

## 4. 选题优先级

椿舍选题排序固定看这五项：

1. `search_value`
2. `emotion_density`
3. `rule_fit`
4. `content_novelty`
5. `conversion_fit`

记住一句话：

`选题看搜索，叙事看人心。`

## 5. 去重规则

- 同一关键词可以复用，但不能连续走同一题角。
- 同一批里，不允许重复：
  - 开头问题
  - 冲突瞬间
  - 主规矩
  - 收口功能
- 默认 7 题只是兜底，不是主生产来源。
