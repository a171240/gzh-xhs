# 椿舍门店专用

## 何时使用
- 用户要为椿舍门店生成小红书短视频文案。
- 用户要按 `李可 / 孟欣 / 伍霖` 三种老板声线写同一批题。
- 用户要把旧版“封面提示词 + 长流程拆解”收成精简发布版。

## 默认调用
- `椿舍内容：{关键词}`
- 可选参数：`角色=李可|孟欣|伍霖`
- 可选参数：`输出=精简发布版`
- 可选参数：`批量=7`

默认值：
- `角色=李可`
- `输出=精简发布版`
- `模式=短视频文案`

## 读取顺序
1. 先读同目录下的 [内容策略.md](./内容策略.md)，它是唯一业务真相。
2. 需要校准结构时，再读 [样例输出.md](./样例输出.md)。
3. 选题或替题时，再查 `关键词包.xlsx`，优先 `主线2：体验转化`。
4. 需要调用金句时，先读 `03-素材库/金句库/00-索引.md`，再只读一个最匹配的主题文件。
5. `椿舍获客Skill.md` 与 `Agent主提示词.md` 只作历史参考，不作默认执行规范。

## 内部执行流程
1. 先把关键词归成 `项目确认 / 避坑情绪 / 问题解决 / 本地确认` 之一。
2. 每篇只选一个具体冲突瞬间，不允许一篇里讲两套冲突。
3. 每篇只选一个老板规矩和一个可感知细节，避免把正文写成服务单。
4. 每篇默认用 2 句金句，必要时才到 3 句；先写正文主张，再选金句轻改，不反过来。
5. 生成 `标题 / 正文 / 置顶评论 / 回复模板 / QC`。
6. 批量时按顺序一篇一篇完成，再做一次重复检查。

## 正文硬标准
- 短视频口播化，默认 `220-320` 字，最高不超过 `380` 字。
- 前 3 句必须能抓停留，优先场景句或判断句，不先讲抽象概念。
- `我是开美容院的` 这类身份锚点只允许在开头出现一次，后文不重复自报身份。
- 每篇最多写 1 个服务流程信息点，最多写 1 个门店细节。
- 正文不出现 CTA，不出现封面提示词，不展示“第 6 步”之类过程说明。
- 正文禁写：`评论 / 私信 / 关注 / 加V / 大众点评 / 团购 / 价格 / 预约 / 到店`。

## 金句调用规则
- 只从一个主题文件里选句，不全库乱抽。
- 优先轻改，不重写；如果轻改后仍然生硬，直接换句。
- 每次改写都检查三件事：
  - 对象不变：顾客不能改成黑头、毛孔、皮肤。
  - 逻辑不变：`不是A，是B` 这类骨架不能丢。
  - 情绪方向不变：安抚、提醒、边界感不能改跑。
- 分布规则：
  - 1 句用于开头钩子或前半段转折。
  - 1 句用于结尾收口。
  - 第 3 句只有在不造成解释重复时才允许放中段。

## 批量稳定规则
- 批量生成 7 篇时，不允许一次性并行起正文，必须顺序完成。
- 默认题组：
  - `去美容院做面部清洁有用吗`
  - `美容院去黑头有用吗`
  - `面部清洁美容院怎么选`
  - `小气泡清洁，最怕做完脸更干`
  - `美容院套路最狠的那一秒`
  - `毛孔粗大怎么改善，先别乱清洁`
  - `吴江美容院先看哪三条`
- 每批固定检查 5 项：
  - 开头句型是否重复
  - 冲突瞬间是否重复
  - 老板规矩是否重复
  - 服务细节是否重复
  - 结尾金句功能是否重复
- 同一批限制：
  - 相同句式开头最多 2 次
  - 相同门店细节最多 2 次
  - 同一条金句原意最多复用 2 次
  - `不是……是……` 结构最多 2 次

## 人设层
- `李可`：更直接、更有规矩感。
- `孟欣`：更柔和、更安抚、更照顾情绪。
- `伍霖`：更利落、更锋利一点，但不攻击人。

三种角色共用：
- 同一批事实
- 同一批选题
- 同一套合规边界
- 同一套金句改写标准

三种角色只允许变：
- 开头口气
- 中段刀口轻重
- 结尾句的温度

## 输出格式
按这个顺序交付：
- 标题
- 正文
- 置顶评论
- 回复模板
- QC

默认不输出：
- 封面提示词
- 全流程策略拆解
- 金句来源说明
- 评论区操盘长段说明
