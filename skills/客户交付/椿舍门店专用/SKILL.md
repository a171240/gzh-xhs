# 椿舍门店专用

## 何时使用
- 用户要为椿舍门店生成小红书短视频文案。
- 用户要按 `李可 / 孟欣 / 伍霖` 三种老板声线写同一批题。
- 用户要把旧版“封面提示词 + 长流程拆解”收成精简发布版。
- 用户要让内容按 `关键词意图 -> 具体人 -> 防御心态 -> 向往画面 -> 门店规矩 -> XHS短视频输出` 的链路稳定出稿。

## 内部执行流程
1. 先跑本地优先的选题引擎：`topic_planner -> topic_dedupe -> topic_pick`。
2. 关键词先归成 `问题修复 / 信任怀疑 / 放松养护 / 本地找店` 之一；若命中高边界词，先做边界处理。
3. 根据意图决定主叙事重心：
   - `问题修复`：结果入口 + 反套路承接
   - `信任怀疑`：拆防御 + 规则先行
   - `放松养护`：向往画面 + 被照顾感
   - `本地找店`：长期感 + 本地确认 + 规则
4. 根据 `阶段` 或自动推断结果，翻译同一个词在不同人生阶段里的真实买点。
5. 用 `富贵内容方法论` 先还原一句“具体的人”：`谁 + 刚发生什么 + 最怕什么 + 真正想要什么`。
6. 拆开 `表面需求` 和 `真实买点`，正文主张优先落到真实买点，不停在项目说明。
7. 每篇只选 1 个冲突瞬间、1 条情绪链、1 条老板规矩和 1 个托底细节。
8. 按意图套用叙事比例：
   - 默认 `2-5-3`
   - `信任怀疑` 允许 `4-3-3`
   - `本地找店` 允许 `1-5-4`
9. 正文默认走双调用主链：`起稿包(draft_package) -> 润稿包(polish_package)`。
10. `快速` 模式默认 2 次模型调用；`平衡` 模式允许 1 次轻量修补重跑；`严格` 模式才允许更多次修补。
11. 金句只在 `polish_package` 末段轻注入，默认只放结尾或结尾前一句，允许最终 0 句成稿。
12. 正文内部至少完成一次 `事实 / 感受 / 想象` 三层转译，但最终只保留自然的两层以上表达。
13. 复审时优先删 A 级套路句，再判断连接词是否真有功能；能用动作、细节、时间、场景接上的，不用硬逻辑词。
14. 复审必须过 3 关：朗读测试、理解测试、真实测试。
15. 批量时先扩题，再按顺序一篇一篇完成，再做一次重复检查。
16. 默认 7 题只兜底，不是主生产来源。

## 正文硬标准（满分稿标准）
- 总标准：`像老板坐在店里跟朋友聊天一样说出来的话。`
- `精简发布版` 默认按短视频口播稿处理，目标 `90-140` 字，`5-7` 行短句，适合 `20-35` 秒直接录。
- `解释型口播版` 目标 `140-220` 字，`7-10` 行短句，可以多一层解释，但仍然要像人说话。

### 故事驱动写法（必须遵守）
1. **开头是一个具体的人的具体经历**：”上周有个姑娘””前两天有个顾客”，不是”很多人搜这个词”。
2. **向往感占正文 50% 以上**：具体瞬间，如”她睡着了””今天挺舒服的””这一个小时没人找你”。
3. **老板不立规矩，老板回应**：”行，你先闭眼，做完叫你”而非”我店里有条规矩”。
4. **叙事比例 2-5-3**：20% 冲突 + 50% 向往画面 + 30% 规矩细节托底。
5. **用”在我这”不用”在椿舍”**，用”瞅了一眼”不用”查看了一下”——老板的下意识用词。
6. **结尾是低标准承诺**：”没白跑就行””别觉得受委屈”，不是”欢迎体验”。
7. **零个”不是A，是B”对比鸡汤句**。

### 正文限制
- 每篇最多写 1 个服务流程信息点，最多写 1 个门店细节。
- 每篇最多只跑 1 条情绪链，比如 `警惕->安心`、`委屈->被尊重`、`尴尬->松一口气`。
- 正文不出现 CTA，不出现封面提示词。
- 正文禁写：`评论 / 私信 / 关注 / 加V / 大众点评 / 团购 / 价格 / 预约 / 到店`。
- 默认不把 `点痣 / 祛斑 / 产后修复 / 盆底 / 身体管理 / 艾灸 / 头疗 / 医美强对照` 直接写成椿舍成交稿；命中这些词时，优先写边界与选择标准。

### 禁令（5 条核心）
1. 禁”不是A，是B”对比鸡汤句（一个都不要）。
2. 禁”我是开美容院的””我店里有条规矩”开头。
3. 禁正文 CTA（大众点评/私信/链接/评论区）。
4. 禁大词堆情绪（绝对不是/本质上/真正的/往往是）。
5. 禁空泛安抚结尾（”希望每个人都能被温柔对待”之类）。

### 更接近富贵原理的写法
- 信任/怀疑词：先写“她为什么会在这个时候搜这个词”，再写“她最怕被怎么对待”。
- 问题修复词：先写“她为什么急着处理这个问题”，再写“她不想再为这个问题继续试错”。
- 放松养护词：先写“她今天为什么撑不住了”，再写“她想被怎样对待”。
- 本地找店词：先写“她为什么不想再白跑一次”，再写“什么样的店让她第二次还敢来”。
- 每篇至少带 1 个生活后果：见人、下班、加班、约会、通勤、开会、回家、周末，不只写项目后果。

## 金句调用规则
- 只有当 `是否调用金句库=是` 时才允许引用或轻改金句。
- 必须使用已加载主题文件中的金句，不得编造“像金句一样的话”。
- 如果启用了金句库但没有主题文件，视为失败，不允许假装已经调用。
- 如果顾客原话已经足够直接，优先用差评原话，不要为了“像金句”把话改虚。
- 只从一个主题文件里选句，不全库乱抽。
- 优先轻改，不重写；如果轻改后仍然生硬，直接换句。
- 每次改写都检查三件事：
  - 对象不变：顾客不能改成黑头、毛孔、皮肤。
  - 逻辑不变：`不是A，是B` 这类骨架不能丢。
  - 情绪方向不变：安抚、提醒、边界感不能改跑。
- 分布规则：
  - 1 句用于开头钩子或前半段转折。
  - 1 句用于结尾收口。
  - 第 3 句只有在不造成解释重复时才允许放中段。
- 椿舍默认最多用 `1-2` 句，重点是“借力”，不是“贴金句标签”。

## 批量稳定规则
- 批量生成 7 篇时，不允许一次性并行起正文，必须顺序完成。
- 默认题组：
  - `去美容院做面部清洁有用吗`
  - `美容院去黑头有用吗`
  - `面部清洁美容院怎么选`
  - `小气泡清洁，最怕做完脸更干`
  - `美容院套路最狠的那一秒`
  - `毛孔粗大怎么改善，先别乱清洁`
  - `吴江美容院先看哪三条`
- 每批固定检查 5 项：
  - 开头句型是否重复
  - 冲突瞬间是否重复
  - 老板规矩是否重复
  - 服务细节是否重复
  - 结尾金句功能是否重复
- 同一批限制：
  - 相同句式开头最多 2 次
  - 相同门店细节最多 2 次
  - 同一条金句原意最多复用 2 次
  - `不是……是……` 结构最多 2 次

## 人设层
- `李可`：更直接、更有规矩感。
- `孟欣`：更柔和、更安抚、更照顾情绪。
- `伍霖`：更利落、更锋利一点，但不攻击人。

三种角色共用：
- 同一批事实
- 同一批选题
- 同一套合规边界
- 同一套金句改写标准

三种角色只允许变：
- 开头口气
- 中段刀口轻重
- 结尾句的温度

## 输出格式
默认按这个顺序交付：
- 标题
- 备选标题（2个）
- 正文
- 置顶评论（本地模板后处理）
- 回复模板（本地模板后处理）

默认不输出：
- 封面提示词
- 全流程策略拆解
- 金句来源说明
- 评论区操盘长段说明
- 审稿过程

只有用户明确要求调试或 review 时，才额外补：
- QC
- 审稿说明
