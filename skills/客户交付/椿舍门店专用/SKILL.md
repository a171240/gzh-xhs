# 椿舍门店专用

## 何时使用
- 用户要为椿舍门店生成小红书短视频文案。
- 用户要按 `李可 / 孟欣 / 伍霖` 三种老板声线写同一批题。
- 用户要把旧版“20-40 秒短稿”升级成 60 秒或 90 秒深口播。
- 用户要让内容按 `关键词意图 -> 具体人 -> 防御心态 -> 向往画面 -> 门店规矩 -> XHS短视频输出` 的链路稳定出稿。

## 内部执行流程
1. 先跑本地优先的选题引擎：`topic_planner -> topic_dedupe -> topic_pick`。
2. 自动起题顺序固定为：`金句评论#选题 -> 用户显式关键词 -> 椿舍专用选题池 -> 默认7题兜底`。
3. 关键词先归成 `问题修复 / 信任怀疑 / 放松养护 / 本地找店` 之一；若命中高边界词，先做边界处理。
4. 根据意图决定主叙事重心：
   - `问题修复`：结果入口 + 反套路承接
   - `信任怀疑`：拆防御 + 规则先行
   - `放松养护`：向往画面 + 被照顾感
   - `本地找店`：长期感 + 本地确认 + 规则
5. 根据 `阶段` 或自动推断结果，翻译同一个词在不同人生阶段里的真实买点。
6. 用 `富贵内容方法论` 先还原一句“具体的人”：`谁 + 刚发生什么 + 最怕什么 + 真正想要什么`。
7. 拆开 `表面需求` 和 `真实买点`，正文主张优先落到真实买点，不停在项目说明。
8. 每篇只选 1 个冲突瞬间、1 条情绪链、1 条老板规矩和 1 个托底细节。
9. 按意图套用叙事比例：
   - 默认 `2-5-3`
   - `信任怀疑` 允许 `4-3-3`
   - `本地找店` 允许 `1-5-4`
10. 正文默认走双调用主链：`起稿包(draft_package) -> 润稿包(polish_package)`，内部骨架固定为：`顾客原话起手 -> 美容师动作推进 -> 情绪变化 -> 男老板判断收口`。
11. `draft_package` 只生成口播节拍，不先写解释型策略稿；旧的 `first_conflict_line / translation_line / boss_judgment_line / low_pressure_offer_line / landing_line` 继续留在 source pack 里当候选素材，但不直接拼正文。
12. `polish_package` 只负责把节拍落成能直接录的视频口播，不自动往正文硬插主题句、老板句、向往句、低压句；缺节拍就重写，不补句子。
13. 男老板视角写死：老板只做 `定规矩 / 转述观察 / 收一句判断`；美容师负责 `水温 / 力度 / 灯光 / 毛巾 / 毯子 / 空调 / 动作观察` 等服务细节。
14. 当题目来自 `quote_topic_seed` 时：至少 1 个标题保留原评论的核心句感，但不能机械照抄；正文最多只留 1 句翻译后的金句判断，放在中段转折或结尾，生硬就降级回标题层。
15. 对标链接只允许贡献 `开头方式 / 推进节奏 / 信任建立逻辑 / 转折结构`，原始句子不能直接进标题和正文。
16. 正文内部至少自然出现两层表达：`事实/动作` + `感受/想象`；不要把“三层表达”写成老师总结。
17. 复审时优先删 A 级套路句，再判断连接词是否真有功能；能用动作、细节、时间、场景接上的，不用硬逻辑词。
18. 复审必须过 3 关：朗读测试、理解测试、真实测试。
19. 批量时先扩题，再按顺序一篇一篇完成，再做一次重复检查。
20. 默认 7 题只兜底，不是主生产来源。

## 正文硬标准（满分稿标准）
- 总标准：`像老板坐在店里跟朋友聊天一样说出来的话。`
- `精简发布版` 默认按 `60秒口播` 处理，目标 `280-360` 字，`10-14` 行短句，适合 `45-65` 秒直接录。
- `解释型口播版` 默认按 `90秒口播` 处理，目标 `420-650` 字，`14-18` 行短句，可以多一层专业翻译或长期信任判断，但仍然要像人说话。

### 故事驱动写法（必须遵守）
1. **前 3 句先把人拉进场景**：必须完成 `具体的人 + 具体原话/具体场景 + 情绪钩子`，不能先解释“为什么会搜这个词”。
2. **前 6 句必须出现顾客原话**：最好是顾客直接说出来的话，或者能一听就知道是顾客在说的话，不要全是老板单口说明。
3. **中段一定靠美容师动作推进**：至少 2 个服务动作，比如 `问水温 / 调力度 / 拉毯子 / 关照空调 / 手机放旁边 / 肩膀慢慢下来`；不要从“她来了”直接跳到“她做完了”。
4. **男老板只做判断，不亲自服务**：老板负责定规矩、转述观察、补一句判断；不要写成老板亲自敷毛巾、调力度、做护理。
5. **情绪链只走一条**：比如 `警惕 -> 松下来`、`累 -> 安静`、`委屈 -> 被尊重`，不要一篇里乱跳。
6. **抽象判断句最多 1 句**：它只能出现在中段转折或结尾，用来收住前面的情绪，不能放在开头当道理。
7. **结尾是低标准承诺**：`没白跑就行`、`不用先做心理准备`、`这一个小时没被添堵`，不是“欢迎体验”。
8. **零个“不是A，是B”对比鸡汤句**。
9. **批量生成时，开头、结尾、老板判断、服务动作四个位置都不能重复**。同一批 7 篇里，“有个姑娘”最多出现 2 次，同一句结尾最多出现 1 次。

### 正文限制
- 每篇最多写 1 个服务流程信息点，最多写 1 个门店细节。
- 每篇最多只跑 1 条情绪链，比如 `警惕->安心`、`委屈->被尊重`、`尴尬->松一口气`。
- 正文不出现 CTA，不出现封面提示词。
- 正文禁写：`评论 / 私信 / 关注 / 加V / 大众点评 / 团购 / 价格 / 预约 / 到店`。
- 默认不把 `点痣 / 祛斑 / 产后修复 / 盆底 / 身体管理 / 艾灸 / 头疗 / 医美强对照` 直接写成椿舍成交稿；命中这些词时，优先写边界与选择标准。

### 禁令（5 条核心）
1. 禁”不是A，是B”对比鸡汤句（一个都不要）。
2. 禁”我是开美容院的””我店里有条规矩”开头。
3. 禁正文 CTA（大众点评/私信/链接/评论区）。
4. 禁大词堆情绪（绝对不是/本质上/真正的/往往是）。
5. 禁空泛安抚结尾（”希望每个人都能被温柔对待”之类）。

### 更接近富贵原理的写法
- 信任/怀疑词：先写她那一秒怎么防着，再写门店怎么让她不用继续防着。
- 问题修复词：先写她急着处理的那一下不舒服，再写她不想再继续试错的后果。
- 放松养护词：先写她今天为什么累到不想讲话，再写她怎样慢慢松下来。
- 本地找店词：先写她为什么怕白跑，再写什么样的店让她第二次还敢来。
- 富贵的“三层表达”只做内部骨架：正文至少自然出现 `事实/动作` 和 `感受/想象` 两层，不要写成“事实是…感受是…想象是…”。
- 每篇至少带 1 个生活后果：见人、下班、加班、约会、通勤、开会、回家、周末，不只写项目后果。

## 金句调用规则
- 金句调用分 3 层：
  - `topic_seed`：带 `#选题` 的爆款评论/金句，可直接生成选题、标题角度、开头钩子。
  - `theme_only`：高优先主题句，只负责定立场、定情绪、定收口，不直接进正文主句。
  - `body_ready`：差评原话和老板口语，负责正文可播内容。
- `topic_seed` 默认开启，用于起题；`是否调用金句库=是` 只控制正文或标题里是否允许显性引用/轻改原句。
- 命中 `topic_seed` 时，至少 1 个标题保留原评论的核心句感，但不能机械照抄原话。
- 正文里显性金句感表达最多出现 `1` 次，只能放在中段转折或结尾；如果翻成老板口语后仍然生硬，就降级回标题层，不强留原句。
- 如果顾客原话已经足够直接，优先用差评原话，不要为了“像金句”把话改虚。
- 只从一个主题文件里选句，不全库乱抽。
- 对标链接只学结构，不借原句；只有当某句已经沉淀进金句库并带 `#选题` 时，才按 `topic_seed` 规则处理。

## 批量稳定规则
- 批量生成 7 篇时，不允许一次性并行起正文，必须顺序完成。
- 默认题组：
  - `去美容院做面部清洁有用吗`
  - `美容院去黑头有用吗`
  - `面部清洁美容院怎么选`
  - `小气泡清洁，最怕做完脸更干`
  - `美容院套路最狠的那一秒`
  - `毛孔粗大怎么改善，先别乱清洁`
  - `吴江美容院先看哪三条`
- 每批固定检查 5 项：
  - 开头句型是否重复
  - 冲突瞬间是否重复
  - 老板规矩是否重复
  - 服务细节是否重复
  - 结尾金句功能是否重复
- 同一批限制：
  - 相同句式开头最多 2 次
  - 相同门店细节最多 2 次
  - 同一条金句原意最多复用 2 次
  - `不是……是……` 结构最多 2 次

## 人设层
- `李可`：更直接、更有规矩感。
- `孟欣`：更柔和、更安抚、更照顾情绪。
- `伍霖`：更利落、更锋利一点，但不攻击人。

三种角色共用：
- 同一批事实
- 同一批选题
- 同一套合规边界
- 同一套金句改写标准

三种角色只允许变：
- 开头口气
- 中段刀口轻重
- 结尾句的温度

## 输出格式
默认按这个顺序交付：
- 标题1
- 标题2
- 标题3
- 正文
当 `配图=是` 且输出为 `精简发布版 / 解释型口播版` 时，额外输出：
- 每篇 1 个 `.cover.md` sidecar，包含封面提示词、负面提示词、结构说明
- 每批 1 个 `椿舍发布包-YYYYMMDD-N篇汇总.md`

默认不输出：
- 把封面提示词直接塞进主文案正文
- 全流程策略拆解
- 金句来源说明
- 评论区操盘长段说明
- 审稿过程

`标题池 + 配图=是` 时：
- 忽略配图
- 只保留 warning，不生成 `.cover.md` 和发布包

只有用户明确要求调试或 review 时，才额外补：
- QC
- 审稿说明
