## 2. 我确认过的核心事实

### 2.1 女性来做美容，不只是为了解决脸

这件事表面看是在处理：

- 黑头
- 毛孔
- 痘印
- 暗沉
- 补水
- 法令纹

但底层其实在处理六件事：

1. 见人安全感  
明天要见客户、同事、朋友、伴侣，我不想一脸疲态先出场。

2. 状态修复感  
最近太累、太黄、太糙、太垮了，我想先把自己拉回来一点。

3. 被照顾感  
我想有一段时间不用回消息、不用负责、不用一直做决定。

4. 体面感  
我不一定要比别人惊艳，但至少不要看起来被生活打烂了。

5. 掌控感  
当生活失序的时候，美容是少数可以立刻做一点什么的动作。

6. 社交验证  
很多第一次消费不是被广告打动，而是被“朋友说这家不坑”推动的。

### 2.2 差评里骂得最多的，不是“没变美”，而是“被消耗”

从你给的评论表里，我实际跑出来的是：

- 有效评论：`10532` 条
- 其中 `0.5-2.0` 星占 `45.3%`
- `3.0-3.5` 星占 `48.6%`

这很重要，因为说明样本不只是“极端暴怒”，而是大量“没有爆炸，但失望、不想再来”的人。

差评聚类后，最核心的几个矛盾是：

- `推销办卡与隐形消费`：`3443` 条，占 `32.7%`
- `时长缩水与手法敷衍`：`3398` 条，占 `32.3%`
- `环境卫生与舒适度`：`2863` 条，占 `27.2%`
- `清洁不到位与效果不稳`：`2138` 条，占 `20.3%`
- `预约难与沟通低效`：`1936` 条，占 `18.4%`
- `服务态度与被冒犯感`：`1815` 条，占 `17.2%`

这说明顾客离开的真正原因，不是单纯“效果差”，而是：

- 情绪上没被接住
- 过程上没被认真对待
- 结果上还不够确定

### 2.3 即便在差评里，顾客买的也还是“舒服”

高频词里，我认为最值得记住的是这几个：

- `服务`：`2832`
- `推销`：`2428`
- `环境`：`1879`
- `手法`：`1666`
- `项目`：`1663`
- `效果`：`1482`
- `态度`：`1399`
- `预约`：`1345`
- `清洁`：`1111`
- `办卡`：`1090`
- `舒服`：`1032`
- `放松`：`442`

这个顺序说明了一件很关键的事：

**顾客评价美容院时，先评价“过程”，再评价“结果”。**

而且哪怕是在差评里，`舒服` 依然被提到 `1032` 次。  
所以美容院不是不该卖“舒服”，而是行业经常把“舒服”做丢了。

### 2.4 搜索端买的是问题，成交端卡在怀疑

从词包和 3.0 执行包反拆后，最稳的结构其实不是“年龄词”，而是四类入口：

1. 问题修复词  
如 `去黑头`、`毛孔粗大怎么改善`、`祛痘印`、`美白`、`闭口怎么去掉`

2. 信任/怀疑词  
如 `美容院做脸有用吗`、`美容院补水项目有用吗`、`美容院套路`、`美容院退费`

3. 放松养护词  
如 `按摩`、`SPA`、`肩颈按摩`、`头部按摩`

4. 本地找店词  
如 `吴江美容院`、`苏州美容院`、`美容院推荐`

我自己的判断是：

- `问题词` 负责流量入口
- `怀疑词` 负责拆防御
- `放松词` 负责建立向往
- `本地词` 负责最后成交

### 2.5 年龄不是搜索入口，但仍然是后台翻译维度

这一点我和 2.0 的看法是同方向，但表述更收敛：

- 前台用户不会搜“30岁女人适合做什么美容”
- 她会搜“毛孔粗大怎么改善”“做脸有用吗”“美容院会不会推销”

但后台做内容和产品包装时，年龄/人生阶段依然有用。  
因为不同阶段，虽然搜的是同一个问题，背后的情绪并不一样。

比如同样搜“补水”：

- 18-24 岁更像“明天别翻车”
- 25-29 岁更像“别又黄又累”
- 30-34 岁更像“别让我再试错”
- 35+ 更像“别让我看起来这么疲惫”

所以我的结论是：

**年龄不该做前台入口，但必须做后台翻译。**

## 3. 我自己的总判断

如果把这轮调研压成一句话：

**顾客搜索时，是冲着效果和状态来的；顾客留下来，却取决于你有没有让她在服务过程中不再被消耗。**

这句话背后有三层：

1. 她不是来买项目名的  
她是来买“状态答案”的。

2. 她不是天然不信任美容  
她是被行业长期训练成了高度防御。

3. 她最后复购，不是因为这次卖得漂亮  
而是因为这次她没有被冒犯、没有被算计、没有更累。

## 4. 我认为最该沉到 skill 里的规则

这里不讲洞察，只讲以后调用时最稳的规则。

### 4.1 入口规则

每次生成内容，先判断关键词属于哪类：

- 问题修复词
- 信任/怀疑词
- 放松养护词
- 本地找店词

不能不分类直接写。

### 4.2 具体人规则

每次都要补完这四件事：

- 她是谁
- 她刚发生了什么
- 她最怕什么
- 她真正想要什么

如果这四个问题答不出来，内容就会滑回空泛介绍。

### 4.3 叙事规则

我认为最稳的结构还是：

- `20%` 刺痛：把她怕的那一秒写出来
- `50%` 向往：让她看见“原来可以这样被照顾”
- `30%` 规矩：用门店边界和细节把信任立住

也就是：

**坑点是入口，向往是成交理由，规矩是托底。**

### 4.4 前台表达规则

不要优先卖这些：

- 项目原理
- 仪器名
- 参数
- 套餐名

要优先卖这些：

- 不会怎么坑你
- 你这次最想解决什么
- 今天适不适合做
- 做完合理能期待什么
- 不适合我会直接说

### 4.5 选题规则

从我的调研看，椿舍这类门店最该优先拍的是四类内容：

1. 高搜索问题词 + 反套路承接
2. 本地词 + 明确规则
3. 放松词 + 轻向往
4. 怀疑词 + FAQ 解释

## 5. 我对 1.0 / 2.0 / 3.0 的最终取舍

### 1.0 该保留什么

- 年龄与人生阶段的情绪翻译
- 美容不是卖项目，是卖体面、掌控、修复、被看见

### 2.0 该保留什么

- 用户真实路径是 `问题 -> 怀疑 -> 到店前防御 -> 进店先判断会不会被套路`
- 第一战不是卖项目，是拆防御

### 3.0 该保留什么

- 词包分组
- 项目包装
- 选题库
- 落地页结构
- 门店话术
- 成交链路

### 我的取舍结论

我自己的版本不是再写第四份大报告，而是把三份里最有用的东西压成一句话：

**1.0 负责讲人，2.0 负责讲路径，3.0 负责讲怎么干。**

而我这份整理，负责把它们变成以后可以稳定调用的规则。

## 6. 最终结论

对椿舍这类门店，我现在最确定的不是“哪个词最爆”，而是这三条：

1. 不要只写避坑  
只写坑，会把账号写成吐槽号。

2. 不要只写向往  
只写梦，会让用户觉得你也像同行一样飘。

3. 要先承认她怕，再给她向往，最后用规矩把向往托实  
这才是既能获客、又能成交、还能长期复购的内容结构。

如果后面要正式升级 skill，我建议就按这条链路做：

`关键词意图 -> 具体人 -> 防御心态 -> 向往画面 -> 门店规矩 -> 输出类型`

这也是我这轮调研里，最值得沉淀下来的方法论。
