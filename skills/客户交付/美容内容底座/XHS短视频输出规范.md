# 美容内容底座｜XHS短视频输出规范

## 1. 主输出范围

v2 只保留 3 种输出：

- `精简发布版`
- `解释型口播版`
- `标题池`

不默认输出：
- 置顶评论
- 回复模板
- QC
- FAQ
- 落地页
- 套餐菜单
- 公众号长文

## 2. 输出协议

### 精简发布版
- 标题1
- 标题2
- 标题3
- 正文

正文标准：
- 默认 `280-360` 字
- 目标 `45-65` 秒深口播
- 建议 `10-14` 行短句
- 固定结构：`爆点评论句感/关键词钩子 -> 具体顾客场景 -> 冲突瞬间 -> 老板回应 -> 一条规矩托底 -> 低压收口`

### 解释型口播版
- 标题1
- 标题2
- 标题3
- 正文

正文标准：
- 默认 `420-650` 字
- 目标 `75-95` 秒深口播
- 建议 `14-18` 行短句
- 在精简发布版结构上增加 `过程细节` 与 `一层专业翻译或长期信任判断`
- 解释服务于拆防御和建立信任，不服务于堆专业

### 标题池
- `8-12` 个标题
- 每个标题配 1 句角度说明
- 标题必须标清关键词意图，不做纯创意堆砌

## 3. 固定 8 步生成流程

1. 关键词进入意图路由。
2. 根据意图决定主叙事重心。
3. 根据阶段翻译真实买点。
4. 还原具体人：`谁 + 刚发生什么 + 最怕什么 + 真正想要什么`。
5. 锁定 1 个冲突瞬间。
6. 锁定 1 条情绪链。
7. 按意图套叙事比例：
   - 默认 `2-5-3`
   - 信任/怀疑 `4-3-3`
   - 本地找店 `1-5-4`
8. 输出 `3个标题 + 1篇正文`。

## 4. 正文质量底线

- 每篇都要能回答：
  - 这个词属于哪类意图
  - 这个人最怕什么
  - 这篇真正卖的买点是什么
- 每篇至少出现两层表达：
  - 事实
  - 感受
  - 想象
- 每篇必须符合：
  - `坑点是入口，向往是成交理由，规矩是托底`
- 对标链接只学结构，不借原句。
- 金句评论如果命中 `#选题`，可以成为一级题源，但正文必须翻成门店老板会说的话。

## 5. 批量 7 篇去重

批量时必须检查：

- 开头句型去重
- 冲突瞬间去重
- 老板规矩去重
- 结尾功能去重
- 同一条金句原意去重

同一批限制：
- 相同句式开头最多 `2` 次
- 相同规矩最多 `2` 次
- 相同冲突瞬间最多 `2` 次
- 相同结尾功能最多 `2` 次
- 相同金句原意最多 `2` 次
