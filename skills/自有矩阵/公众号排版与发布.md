# 公众号排版与发布

## 说明
这不是 repo-local 内容 skill，不进入 `skill-manifest.json`。
它是公众号全链路的下游适配说明，实际执行继续走 automation：
- `排版公众号：<md路径>`
- `发布公众号：<md路径>`
- `确认发布：<task_id>`

---

## 真相源
- 已生成的公众号 markdown
- `summary`
- `cover_image`
- `## 正文`
- `## CTA`
- 本地渲染 manifest

---

## 执行链路
1. `排版公众号`
   - 调用本地渲染器
   - 产出 `preview.html` / `clipboard.html` / `render-manifest.json`
2. `发布公众号`
   - 调用 `publish_action_runner.py`
   - 执行 `prepare`
   - 保存草稿与截图证据
3. `确认发布`
   - 对既有任务执行 `approve`
   - 不重新渲染正文

---

## 命令契约
- `排版公众号：<md路径>` -> `preview + manifest`
- `发布公众号：<md路径>` -> automation task，状态通常为 `pending_approval`
- `确认发布：<task_id>` -> automation task 最终状态

---

## 边界
- 内容、图片、排版、发布已经拆成不同阶段
- 发布不是普通内容 skill，不走 `run_skill_task()`
- 如果 markdown 缺少 `summary`、`cover_image` 或正文图片，发布前必须先补齐
