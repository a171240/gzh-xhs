# Skill: 数据复盘 v2.0

## 升级摘要
v2.0 融合数据驱动闭环，核心变化：
- 新增自动评级机制（基于阅读量和在看率）
- 新增标题得分和开头得分评估
- 接入数据反哺脚本（评级≥A自动触发）
- 输出复盘报告（标题得分/开头得分/改进建议/可复用元素）

---

## 触发方式
用户说"记录数据：{平台} {标题} {数据}"

**输入示例**：
```
记录数据：
- 平台：公众号
- 标题：为什么你学了那么多爆款技巧，内容还是做不起来？
- 数据：阅读量6000/在看300/转发50/评论20
```

---

## 执行流程

### Step 1: 记录数据到统计表
将数据追加到 `04-数据与方法论/内容数据统计/公众号数据.md`

**记录格式**：
| 日期 | 账号 | 标题 | 标题公式 | 核心关键词 | 阅读量 | 在看 | 转发 | 评论 | 完读率 | 评级 | 标题得分 | 开头得分 | 备注 |

**字段说明**：
- 标题公式：从标题中识别使用的公式（如"反常识体"、"年龄+状态+反转"）
- 核心关键词：从标题中提取核心关键词
- 评级：自动评级（见评级标准）
- 标题得分：基于标题公式、关键词、情绪词的综合评分（1-10分）
- 开头得分：基于钩子强度、场景细节的综合评分（1-10分）

---

### Step 2: 自动评级

**评级标准**（公众号）：
- **S级**：阅读量>5000 或 在看率>5%
- **A级**：阅读量>2000 或 在看率>3%
- **B级**：阅读量>1000 或 在看率>2%
- **C级**：阅读量<1000

**在看率计算**：在看数 / 阅读量 * 100%

---

### Step 3: 触发数据反哺（评级≥A）

若评级≥A，自动触发数据反哺脚本：
```bash
python 06-工具/scripts/data_feedback_engine.py --platform wechat --min-grade A
```

**数据反哺流程**：
1. 分析S级/A级内容的标题公式、关键词、开头结构
2. 更新 `04-数据与方法论/方法论沉淀/标题方法论.md`（新增"已验证的高效标题模式"）
3. 更新 `03-素材库/关键词库/公众号核心词库.md`（更新"使用次数"和"转化率"）

## 评分标准

### 标题得分（1-10分）
- **标题公式有效性**（3分）：使用已验证的高效公式
- **关键词精准度**（3分）：关键词与用户搜索意图匹配
- **情绪触发点**（2分）：标题能否触发情绪（好奇/焦虑/共鸣）
- **前4字差异化**（2分）：与同期内容不重复

### 开头得分（1-10分）
- **钩子强度**（4分）：前3秒能否抓住注意力
- **场景细节**（3分）：是否有具体的场景描述
- **情绪共鸣**（3分）：是否能引发读者共鸣

**输入**：
```

- 标题：为什么你学了那么多爆款技巧，内容还是做不起来？

```

**输出**：

```
- 发布日期：2026-03-05
- 平台：公众号
- 账号：IP内容工厂
- 标题：为什么你学了那么多爆款技巧，内容还是做不起来？
- 阅读量：6000
- 在看：300（在看率：5%）
- 转发：50
- 评论：20
- 评级：S级

## 📝 标题分析
- 标题公式：反常识体
- 核心关键词：爆款技巧
- 标题得分：9/10分
- 得分依据：
  - 公式有效性：反常识体，已验证高效公式
  - 关键词精准度：精准命中"学了不会用"的痛点
  - 情绪触发点：触发"技巧陷阱"的焦虑情绪

- 开头得分：8/10分
- 得分依据：
  - 钩子强度：数据钩子"90%的人"，吸引力强
  - 场景细节：有具体的场景描述

## 🔄 可复用元素
- 标题公式：反常识体（已验证有效，可复用）
- 开头结构：数据钩子+场景描述（已验证有效，可复用）

✅ 已触发数据反哺脚本
✅ 已更新标题方法论

```
