# 公众号四账号内容生成

## 适用场景
- 直接触发：`生成公众号内容：<Brief>`
- 强触发：`/skill wechat 需求: <Brief>`
- 默认交付模式固定为四账号批量：`gongchang / ipgc / zengzhang / shizhan`

这是公众号内容主 skill，只负责：
1. 解析 Brief
2. 生成四账号标题
3. 生成四账号大纲
4. 生成四账号正文
5. 按 `FILES_JSON + FILE_BLOCK` 打包交付

它不负责直接出图、排版或发布。

## Brief Schema

```text
日期(YYYY-MM-DD，可空，默认今天)：
主题矿区/选题：
目标人群：
核心矛盾：
场景证据/案例素材：
希望读者做的第一步：
引流物(CTA关键词)：
禁区(可选)：
是否调用金句库：是|否
金句主题：
参考对标文案：
富贵模块开关：是|否
```

字段缺失处理：
- 可以补全假设，但必须在内部先补齐逻辑，不要反复追问废确认
- 不得编造来源、数据、案例
- 若 `参考对标文案` 已提供但分析报告缺失，先补分析再继续

## 核心规则

### 1. 四账号必须明显区分
- `gongchang`：系统方法论、框架、机制、底层逻辑
- `ipgc`：情绪共鸣、人生阶段、关系张力、情绪反转
- `zengzhang`：数据增长、样本洞察、指标语言、增长杠杆
- `shizhan`：轻实操、避坑、清单、马上能做

### 2. 标题必须互斥
- 四个主标题必须使用不同标题公式
- 四个主标题前 4 个字不能相同
- 标题以 18-28 字为优先，尽量少标点

### 3. 金句库必须受开关控制
- 只有当 `是否调用金句库：是` 时才允许引用/改写金句
- 只能使用已加载主题文件中的金句，不得编造“像金句一样的话”

### 4. 对标文案必须先经过分析
- `参考对标文案` 不是直接照抄素材
- 只能消费分析报告中的：类型、结构、情绪触发点、可复用金句、案例、框架
- 仍然要改写成四账号差异化内容

### 5. 正文必须通过交付前质检
- 每篇 1500-3000 字，信息密度优先
- 每 2-4 行换行
- 至少 1 句可截图金句（建议加粗）
- 至少 2 处反驳安全阀
- 至少 3 处埋点（数字对比 / 反常识 / 场景细节）
- 禁止空洞鸡汤、假共识、强 AI 套话

## 工作流

### Step 0：预检
- 校验 Brief 字段
- 校验条件上下文是否存在
- 如果启用了金句库但缺少主题文件，直接失败
- 如果给了 `参考对标文案` 但找不到分析报告，先补分析或直接失败

### Step 1：四账号标题
每个账号输出：
- 主标题 1 条
- 备选标题 2 条
- 标注标题公式

### Step 2：四账号大纲
在四账号大纲前，先输出一份统一目标读者画像。

每个账号大纲必须包含：
- 开头四件套：身份锚点 / 场景细节 / 扎心矛盾 / 钩子承诺
- `01 / 02 / 03` 主体结构
- 每点至少包含：观点 / 案例细节 / 底层逻辑 / 行动第一步 / 反驳安全阀
- 结尾承接 `CTA`

### Step 3：四账号正文
每篇必须体现账号语气差异，不得同文改词。

推荐正文结构：
- 开头：高信息密度切入
- 主体：`###` 小节展开
- 文末：明确动作和 CTA

### Step 4：打包交付

#### 路径规则
- 目录：`02-内容生产/公众号/生成内容/YYYY-MM-DD/`
- 文件：`{prefix}-{YYYYMMDD}-{短标题}.md`

#### 每个 md 文件必须包含
- YAML frontmatter
  - 至少包含：`账号`、`日期`、`选题`、`标题公式`、`配图风格`、`summary`
  - `summary` 必须是 80-140 字中文摘要
- `## 标题`
- `## 正文`
- `## CTA`
- `## 配图提示词`

#### `## 配图提示词` 约束
- 只要求提供“语义稿”，不要求在本 skill 内完成标准化
- 至少包含：1 张封面 + N 张正文图语义说明
- 后续由 `公众号配图提示词标准化` 统一改写成正式 bullet 契约

## 输出合同

除以下结构外，不要输出任何多余文字。

### 1) FILES_JSON
```text
<!--FILES_JSON_START-->
{"date":"YYYY-MM-DD","files":[{"path":"02-内容生产/公众号/生成内容/YYYY-MM-DD/gongchang-YYYYMMDD-短标题.md"},{"path":"02-内容生产/公众号/生成内容/YYYY-MM-DD/ipgc-YYYYMMDD-短标题.md"},{"path":"02-内容生产/公众号/生成内容/YYYY-MM-DD/zengzhang-YYYYMMDD-短标题.md"},{"path":"02-内容生产/公众号/生成内容/YYYY-MM-DD/shizhan-YYYYMMDD-短标题.md"}]}
<!--FILES_JSON_END-->
```

### 2) FILE BLOCK
```text
<!--FILE_START-->
02-内容生产/公众号/生成内容/YYYY-MM-DD/gongchang-YYYYMMDD-短标题.md
<完整 md 内容>
<!--FILE_END-->
```

共输出 4 个文件块，顺序必须和 JSON 一致。
