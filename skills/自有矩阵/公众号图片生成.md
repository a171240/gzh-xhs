# 公众号图片生成

## 定位
这是执行型 skill，只负责调用图片脚本完成出图、压缩、回写 markdown。

它不是知识库融合器，也不是提示词重写器。

标准链路固定为：
1. 先运行 `公众号配图提示词标准化`
2. 再运行 `公众号图片生成`

---

## 触发方式
- `生成公众号图片：2026-03-06`
- `生成公众号图片：02-内容生产/公众号/生成内容/2026-03-06/gongchang-20260306-示例.md`
- `/skill 公众号图片生成 需求: 2026-03-06`

---

## 唯一执行真相源
- `06-工具/scripts/README_WECHAT_IMAGE_GENERATOR.md`
- 当前目标文章 markdown

图片脚本读取的输入真相源是文章 markdown 里已经标准化完成的 `## 配图提示词`。

如果文章里还是旧格式、弱提示词、缺封面图、缺章节图，不要在这个 skill 里临时猜测或重写；应该先返回失败，并要求先执行 `公众号配图提示词标准化`。

---

## 输入约定
- 输入只能是两类之一：
  - 日期：`YYYY-MM-DD`
  - 单个 markdown 路径：仓库内的 `.md` 文件路径
- 可透传的执行参数：
  - `--max-size-kb <数字>` 或 `max-size-kb=<数字>`
  - `--no-insert` / `不回写`
  - `--no-compress` / `不压缩`

---

## 前置校验
1. 如果输入是日期：
   - 目录必须存在：`02-内容生产/公众号/生成内容/<日期>/`
   - 目录下至少有 1 个 `.md` 文件
2. 如果输入是路径：
   - 文件必须存在
   - 文件必须位于当前仓库内
3. 每个目标 md 都必须满足：
   - frontmatter 至少包含 `账号`
   - 存在 `## 标题`
   - 存在 `## 正文`
   - 存在 `## 配图提示词`
4. `## 配图提示词` 必须已经是标准 bullet 契约：
   - `- 封面图：...`
   - `- 配图1（对应：...）：...`
   - `- 配图2（对应：...）：...`
5. 如果缺少上述任一条件，直接失败，不要继续调用脚本。

---

## 执行规则
执行命令固定使用：

```bash
python 06-工具/scripts/wechat_image_generator.py --batch-date <日期> --insert-to-md --compress
python 06-工具/scripts/wechat_image_generator.py <md路径> --insert-to-md --compress
```

附加规则：
- 若输入显式要求 `不回写` / `--no-insert`，则改为追加 `--no-insert`
- 若输入显式要求 `不压缩` / `--no-compress`，则改为追加 `--no-compress`
- 若输入中给了 `max-size-kb`，把它透传给脚本
- 不要自己手工拼接图片 markdown
- `cover_image` 回写、正文图插入、`index.json` 写入统一交给 `wechat_image_generator.py`

---

## 成功输出
只返回简洁执行摘要，不输出 markdown 文件块。

必须说明：
- 执行的是批量模式还是单文件模式
- 实际命令
- 处理了哪些 md
- 是否已回写 `cover_image` 和正文图片引用
- 图片产物目录（`images/<abbr>/`）
- 若有失败项，逐条列出

建议格式：

```text
状态：成功
模式：批量/单文件
命令：python ...
处理文件：
- 02-内容生产/公众号/生成内容/...
结果：
- 已生成 images/gongchang/cover.jpg
- 已回写 cover_image 与正文插图
```

---

## 失败输出
直接说明失败原因和对应路径。

常见失败原因：
- 日期目录不存在
- 目录下没有 md 文件
- frontmatter 缺少 `账号`
- 缺少 `## 正文`
- 缺少 `## 配图提示词`
- `## 配图提示词` 未标准化，缺少 `封面图 + 配图1..N`
- 图片生成重试后仍失败
- 启用压缩但未达到 `max-size-kb`

不要输出空泛道歉，不要编造“已成功”。
