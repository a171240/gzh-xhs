# archive 使用说明（冻结区）

`archive/` 为历史归档目录，默认只读，不参与当前主生产流程。

## 归档规则

- 仅用于追溯历史方案、旧提示词、旧产物。
- 不在日常“选题 -> 生产 -> 发布 -> 复盘”链路中自动检索。
- 非必要不改写历史文件；若需修订，仅做最小修复（断点、说明、索引）。

## 与新结构的映射（常见）

- `archive/公众号内容生成器/` -> `02-内容生产/公众号/` + `skills/自有矩阵/公众号批量生产.md`
- `archive/小红书内容生成/` -> `02-内容生产/小红书/` + `skills/自有矩阵/小红书内容生产.md`
- `archive/金句库_原始/` -> `03-素材库/金句库/`
- `archive/生成内容/` -> `01-选题管理/03-已发布/`（按平台/日期归档）

## 使用建议

- 若要复用 archive 内容，先复制到新结构对应目录再编辑。
- 避免在 archive 内直接新增“主流程文件”，防止路径混乱。
