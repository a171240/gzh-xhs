# 小红书双模内容生产线（XHS）

目标：把同一个 Brief 路由到两种可发布的交付包。

双模：
1. `信息图6页`：封面 + P2-P5 展开 + P6 行动引导
2. `情绪冲突文字帖`：正文不出现 CTA；把互动与轻 CTA 放在置顶评论

## 快速开始（输入 Brief）

优先让用户按这个结构给你 Brief（字段缺失就先补问，别猜）：

```text
模式：信息图6页|情绪冲突文字帖
账号角色：A/B/C（或直接写：账号定位/语气/人设）
主关键词：
目标人群：
核心目标：引发收藏/评论/私信
置顶互动开关：ON|OFF
是否调用金句库：是|否
金句主题：搞钱与生意/内容与增长/系统与执行/人性与沟通
禁区(可选)：不聊什么/不碰什么
补充素材(可选)：你自己的经历/数据/截图要点
```

## 工作流（严格按模式分支）

### Step 0：加载上下文（按需，不要一股脑全塞）

必选（如果路径存在）：
- `小红书内容生成/小红书内容生成Skill_完整版.md`
- `小红书内容生成/resources/高情绪标题公式.md`
- `小红书内容生成/resources/消费者洞察词库.md`
- `小红书内容生成/resources/证据素材库.md`

可选（仅当你要做更尖锐但合规的情绪/争议帖）：
- `小红书「争议讨论帖」话题生成 Agent（安全尖锐版）.md`
- `小红书内容生成/resources/情绪冲突内容引擎（全行业通用）.md`

可选（仅当 Brief 明确允许）：
- 金句库：`金句库/00-索引.md` + 你选的主题文件（例如 `金句库/03-系统与执行.md`）
- 富贵模块：`富贵-打动人模块.md`（用于“具体的人”“事实/感受/想象”改写）

金句库硬约束：
- 只有当 Brief 写了 `是否调用金句库：是` 才能引用/改写金句。
- 只允许从已加载进上下文的金句库文件里挑选并改写；不要凭空编造“金句”。

### Step 1：解析 Brief 与选题定位

输出：
- 模式识别与要点摘要（<=6 行）
- 主关键词 + 3-6 个长尾词
- 标题候选 3 条（<=20 字，强情绪但合规）
- 标签 10 个（主关键词 + 长尾 + 人群/场景）
- 策略：
  - 信息图：推荐结构类型（流程/层级/对比/因果/矩阵）+ 互动目标
  - 情绪冲突：一句话冲突主张 + 3 个争议点 + 3 个评论区问题

### Step 2：正文/文案生产（按模式）

模式 A：信息图 6 页
- 给 2000 字级“内容结构”（模块1/2/3/4）
- 给 6 页结构（封面 + P2-P5 + P6）
- 每页文案 <=60 字：短句、强对比、可截图

模式 B：情绪冲突文字帖
- 标题 1 条（<=20 字）
- 正文 800-1200 字：段落短；不出现 CTA；结尾只留评论区问题
- 冲突结构：立场句 -> 反方强版本 -> 拆穿 -> 可执行一步 -> 抛问题
- 置顶评论：
  - 仅当 `置顶互动开关：ON` 时输出（互动引导 + 轻 CTA）
  - 否则输出：`置顶评论：关闭`
- 风险自检 5 条（法律/医疗/夸大/群体攻击/人身攻击等）

### Step 3：配图提示词（可直接喂给作图模型）

模式 A：每页都给提示词（封面 + P2-P6）
- 每张图包含：风格、排版、配色、主元素、文字区、负面提示词
- 输出中文提示词 + 英文提示词 + 负面提示词

模式 B：只给封面图提示词（1 张）
- 小红书常见极简封面：留白 + 大字 + 1 个对立元素（不攻击具体人/群体）
- 输出中文提示词 + 英文提示词 + 负面提示词

### Step 4：交付包与质检（最终输出）

最终输出固定 Markdown 结构（不要夹杂多余解释）：
- `模式：...`
- `标题：...`（3 条）
- `标签：...`（10 个）
- `正文/文案：...`（按模式）
- `置顶评论：...`
- `配图提示词：...`
- `质检清单：...`

## 合规底线（默认开启）

- 不编造数据与案例；不做医疗/法律断言；不做人身攻击；不煽动对立。
- 可以“尖锐”，但必须“安全”：用观点与逻辑对冲，不用群体标签和羞辱。
