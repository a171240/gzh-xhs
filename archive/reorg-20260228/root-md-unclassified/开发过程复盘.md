# 用Claude Code从0到1搭建小红书自动化内容工厂

> 本文记录了我使用Claude Code（Anthropic官方CLI工具）开发小红书自动化内容生成+发布系统的完整过程，包括每一步的提示词和实现细节。

---

## 一、项目背景

### 1.1 目标
为B2B获客搭建一套完整的小红书内容矩阵系统：
- **3个账号**：A号(转化号)、B号(交付号)、C号(观点号)
- **自动生成**：文案 + 8张轮播图
- **自动发布**：Playwright浏览器自动化
- **数据追踪**：飞书多维表格存储

### 1.2 技术栈
- **AI图片生成**：Gemini 3.0 Pro（通过gemini-webapi库）
- **浏览器自动化**：Playwright
- **编程语言**：Python 3.11+
- **开发工具**：Claude Code CLI

### 2.4 第四阶段：Gemini图片生成（重点！）

这是最复杂的部分，经历了多次调试。

**我的提示词：**


**问题**：网页登录状态经常失效，CDP连接不稳定。

#### 第二次尝试：gemini-webapi库


```python
    def __init__(self, secure_1psid: str = None, ...):
        # 从环境变量或参数获取cookies
        self.secure_1psid = secure_1psid or os.getenv("GEMINI_SECURE_1PSID")
        ...

    async def generate_image(self, prompt: str, ...):
        response = await self.client.generate_content(
            full_prompt,
            model=self.model_id
        )
        # 保存图片
        ...
```

## 四、完整使用流程

### 4.1 环境准备

```bash
# 安装依赖
pip install gemini-webapi playwright python-dotenv lark-oapi

# 安装浏览器
playwright install chromium
```

### 4.2 配置Gemini Cookies

1. 打开Chrome，登录 https://gemini.google.com
2. F12 → Application → Cookies → gemini.google.com
3. 复制 `__Secure-1PSID` 和 `__Secure-1PSIDTS` 的值

创建 `crawler/.env` 文件：
```
GEMINI_SECURE_1PSID=你的cookie值
GEMINI_SECURE_1PSIDTS=可选的cookie值
```

### 4.3 生成内容和图片

```bash
cd crawler

# 生成单张测试图
python run_image_generator.py --test -a A

# 生成完整8张图（指定主题）
python run_image_generator.py -a A -t "AI选题工具"

# 只生成封面和P2
python run_image_generator.py -a A --pages 封面 P2
```

### 4.4 自动发布

```bash
# 1. 首次登录（扫码）
python run_xhs_publisher.py login -a A
python run_xhs_publisher.py login -a B
python run_xhs_publisher.py login -a C

# 2. 检查登录状态
python run_xhs_publisher.py check -a all

# 3. 发布笔记
python run_xhs_publisher.py publish -a A -c sample_content.json

# 4. 批量发布（3账号轮转）
python run_xhs_publisher.py batch -i sample_batch.json

# 5. 启动调度器（后台运行）
python run_xhs_publisher.py run
```

## 六、经验总结

### 6.1 Claude Code的优势

1. **自动调研**：使用WebSearch搜索最新资料
2. **代码生成**：一次性生成完整的模块代码
3. **错误修复**：根据错误信息快速定位问题
4. **文件管理**：自动创建目录和配置文件

### 6.2 开发技巧

1. **分阶段开发**：先调研 → 设计 → 实现 → 测试
2. **及时测试**：每完成一个模块就测试
3. **错误反馈**：把完整的错误信息给Claude
4. **迭代优化**：遇到问题不要怕，让Claude换方案

### 6.3 注意事项

1. **Cookies有效期**：Gemini cookies会过期，需要定期更新
2. **频率限制**：发布间隔要足够长，避免账号风险
3. **登录状态**：使用独立浏览器Profile保持登录
4. **内容去重**：3个账号内容要有差异化
