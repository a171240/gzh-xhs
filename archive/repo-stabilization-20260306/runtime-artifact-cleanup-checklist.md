# runtime artifact 清理清单

## 适用范围
本清单只适用于运行产物，不适用于 `01-05` 下的内容资产。

使用前先对照：
- `change-buckets.md`
- `runtime-artifact-governance.md`

## 清理前检查
- [ ] 当前是否有长驻进程正在运行：
  - `feishu_kb_orchestrator.py` daemon
  - `topic_pipeline.py` daemon
  - `ingest_writer_api.py`
- [ ] 当前是否仍需要保留最近一次成功/失败证据样本
- [ ] 目标目录是否属于 `reports/`、`06-工具/data/`、缓存目录，而不是 `01-05`
- [ ] 若涉及 `06-工具/data/profiles/`，是否明确确认“不是正在使用的登录态”

## 默认保留项
这些内容默认不应被“整理仓库”顺手删除：
- `06-工具/data/profiles/`
- 最近一次可复现的发布/排版/抓取成功样本
- 最近一次失败样本及其截图、run log
- 人工整理过的日报、分析报告、方法论文档

## 默认可清理项
满足“已不再用于定位问题或保留证据”时，可以清理：

### 1. `reports/`
可清理对象：
- 过期 smoke 目录
- 重复 dry-run payload
- 已失去定位价值的旧截图
- 重复 preview / render manifest 副本

保留建议：
- 每条主链至少保留 1 份最近成功样本
- 每类近期故障至少保留 1 份最近失败样本

### 2. `06-工具/data/`
可清理对象：
- 已停用任务的旧日志副本
- 过期 heartbeat / state 快照备份
- 无需继续追踪的历史 automation 副本

禁止顺手清理：
- 正在被服务使用的状态文件
- 仍承载登录态的 `profiles/`
- 尚未确认是否被恢复流程依赖的队列/状态文件

### 3. 本地依赖与缓存
可清理对象：
- `node_modules/`
- `__pycache__/`
- `*.pyc`
- 本地 npm / Python 临时缓存

前提：
- 已有 `package-lock.json` 或 requirements/lock 文件
- 可接受后续按锁文件重建

## 不可按 runtime artifact 清理的内容
即使目录名看起来像“生成/日报/报告”，以下内容默认不在本清单范围内：
- `01-选题管理/`
- `02-内容生产/`
- `03-素材库/`
- `04-数据与方法论/`
- `05-业务运营/`

## 建议清理顺序
1. 先清 `reports/` 的重复样本
2. 再清 `06-工具/data/` 下明确无用的日志/副本
3. 最后清 `node_modules/`、缓存目录
4. 不动 `profiles/`，除非明确指定

## 清理后验证
- [ ] `python -m pytest 06-工具/scripts/tests -q` 仍通过
- [ ] 若保留有长驻服务，确认服务仍可启动/仍有有效状态
- [ ] 若涉及发布链路，确认登录态未误删
- [ ] 若删除了调试证据，确认至少仍保留一份最近成功/失败样本

## 原则
- 先判定所有权，再决定删除
- 先保可复现证据，再谈整洁
- 不用“清空目录”换表面干净
