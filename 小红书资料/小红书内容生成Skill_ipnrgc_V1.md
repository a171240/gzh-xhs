# 小红书矩阵内容生成 Skill（基于 ipnrgc.com）
> 目标：用「内容矩阵 + 搜索卡位 + 投放」稳定拿到 **预约诊断**，最终转化为 **会员**。  
> 适用：代运营团队 / MCN矩阵 / 企业内容中台 / 知识付费团队 / 本地连锁内容组（与你站内主打场景一致）。

## 0. 先把真相说清：你现在缺的不是内容，是“能成交的内容工序”
ipnrgc.com 的核心卖点很明确：把「定位 → 选题 → 日历 → 脚本 → 质检」写进可复制的交付线，并且提供 **免费快速诊断（约5分钟、8道题、五维评分+行动清单）** 作为入口，附带 60期内容日历、可拍脚本、质检与去重归档等交付产物示例。  
这些卖点必须被翻译成：**小红书用户一眼能懂的“痛点—解决方案—证据—行动”**。否则你写得再好，也只是自嗨。 citeturn2search0

## 1. Skill 的输入（Brief Schema）
AI 每次生成内容之前，必须拿到这些字段（缺一就容易跑偏）：

### 1.1 基础信息
- **账号角色**：A转化号 / B交付负责人号 / C操盘手观点号  
- **行业/赛道**：代运营 / MCN / 品牌内容部 / 知识付费 / 本地连锁 等  
- **阶段**：冷启动（0-100线索）/ 稳定增长 / 规模化（投放放量）  
- **目标**：预约诊断 / 预约演示 / 会员试用（7天）/ 直接购买

### 1.2 关键词与意图
- **主关键词（1个）**：用于标题与首段（自然融入）  
- **长尾关键词（2-4个）**：用于正文小标题/要点  
- **用户意图标签**：找模板 / 找系统 / 提效 / 对比工具 / 交付质检 / 多账号管理  
- **竞争强度**：高/中/低（决定你写“泛”还是写“窄”）

### 1.3 可用证据资产（没有客户案例也能做）
- 你站内已经有的：**诊断报告样本片段、脚本片段、内容日历片段、质检清单与去重归档**（把这些做成“可截图、可演示”的证据）。 citeturn2search0  
- 你自己可以补的：录屏 Demo（60秒）、流程图、模板截图、前后对比（返工次数/交付节奏变化用“方法论示例”呈现，别虚构客户）

### 1.4 合规约束（必须写进 Skill）
- 小红书在社区规范中对“交易及导流行为”有约束，不鼓励发布营销/导流信息。 citeturn1search0  
- 交易导流规则强调应使用商城、购物车、**官方留资组件**等方式，避免引导站外交易。 citeturn1search7turn1search9turn1search0  
> 结论：**你的CTA必须优先用“评论关键词/私信/官方留资组件”承接，别在笔记/评论里硬塞微信号。**（网站可以放，但小红书内容里不要赌命。）

## 2. Skill 的三条铁律（写进系统提示词里）
1) **证据优先**：每篇笔记至少出现 1 个“可验证证据”（截图/录屏/样本片段/清单）。没有证据的观点一律降权。 citeturn2search0  
2) **单篇单目标**：一篇笔记只做一件事：引发“评论/私信拿资料”或“做诊断”。别一篇里又卖课又卖系统又讲故事。  
3) **关键词=需求**：标题不是文学，是用户搜索的入口。SEO笔记必须让关键词自然露出，别堆砌。

## 3. Skill 工作流（最短路径，不牺牲转化）

### Step 1：选“账号角色 + 目标阶段”
- **A转化号**：围绕诊断/演示/会员试用（唯一收口）  
- **B交付号**：围绕“交付稳定”“口径一致”“质检去重”“内容日历”  
- **C观点号**：围绕“为什么你们越做矩阵越累”“内容团队效率真相”“流程化 vs 灵感化”

### Step 2：选“关键词簇 + 搜索意图”
把关键词分成4类（你已经有词库，这里是用法）：  
- 高热高转化：直接做“解决方案/对比/工具清单”  
- 高热低转化：做“引流干货”，末尾引到诊断  
- 低热高转化：做“专业痛点”，强势卡位（多账号、质检、内容中台）  
- 冷门无效：不写、不投

### Step 3：确定“笔记类型”（决定结构）
只用这 6 种，够你跑一年：
1) **问题-解决型**（最快转化）  
2) **清单模板型**（最高收藏）  
3) **SOP流程图型**（最强信任）  
4) **对比评测型**（最高意图）  
5) **避坑复盘型**（最高共鸣）  
6) **Demo演示型**（最强证据）

### Step 4：组装“论证骨架”（固定顺序）
- 痛点（具体场景）  
- 机制（为什么会这样）  
- 方法（你给的步骤/清单）  
- 证据（样本/截图/录屏）  
- 行动（评论关键词/做诊断）  

### Step 5：生成内容（输出三件套）
每次生成必须输出：
- **标题 5条**（含主关键词，分别：痛点/数字/反直觉/对比/结果）  
- **正文 1篇**（700-1200字区间为主，段落短，列表化）  
- **封面文案 2条 + 话题标签 10个**（含长尾词）

### Step 6：生成“承接话术”（不靠微信）
- 评论区置顶回复（引导私信关键词）  
- 私信自动回复（发资料/发诊断入口/引导预约演示）  
- 二次追问（筛选B端：团队规模、账号数量、交付流程）

> 若你使用聚光做“客资收集”，把 CTA 统一改成“点击留资/领取诊断清单”。聚光官方将“客资收集”列为营销场景之一。 citeturn0search12turn0search4

### Step 7：质检与去重（矩阵账号的生死线）
- **去重**：同一关键词簇，三账号必须“不同角度、不同结构、不同开头钩子”。  
- **低质矩阵风险**：平台对大量低质量、同质化矩阵内容有打击趋势，别用模板批量糊。 citeturn1search10turn1search12  
- **夸大/虚假**：不写“保证涨粉/保证变现”，不虚构客户案例。

## 4. 单篇笔记模板（可直接让AI填空）

### 模板A：问题-解决（强转化）
**标题**：`{主关键词} 做不起来？90%的人卡在 {卡点}`  
**开头3秒**：一句扎心现状 + 你给结果（不是鸡汤）  
**主体**：
- 01 你以为的问题，其实是 {底层机制}  
- 02 正确做法：{3步SOP}  
- 03 交付里必须有的3个产物：{日历/脚本/质检} citeturn2search0  
**证据**：贴“诊断报告/脚本/日历”样本片段（图或文字节选） citeturn2search0  
**CTA**：`评论【诊断】我发你8题五维评分+30天行动清单` citeturn2search0

### 模板B：清单模板（高收藏）
**标题**：`{主关键词} 模板（可直接复制），我把它拆成 12 条清单`  
**主体**：12条清单 + 1张“质检表”截图  
**CTA**：`私信【模板】领取可编辑版`

### 模板C：SOP流程图（强信任）

## 5. 搜索型关键词 vs 投放型关键词（怎么用）

### 5.1 搜索型关键词（SEO卡位）
写法：标题=主关键词；首段=主关键词+长尾；小标题=长尾；话题=长尾。  
目标：让笔记在搜索结果里长期吃流量。

### 5.2 投放型关键词（聚光/信息流定向）
写法：你投的不是“词”，是“人群意图”。  
- 冷启动：用高热词拉量 + 内容钩子收线索  
- 稳定期：用低热高转化词提高线索质量（代运营交付、内容中台、矩阵管理）  
聚光平台支持多营销场景（含“客资收集”），并且有搜索+信息流的组合能力。 citeturn0search12turn0search4

### 5.3 薯条怎么用（只放大“已验证内容”）
薯条服务属于“有偿内容加热/营销推广服务”，适合把自然数据好的笔记继续加热，不适合拿来救垃圾内容。 citeturn0search1turn0search5

## 6. 数据复盘（只看三类指标）
别被点赞骗了，你要看：
1) **收藏率/私信触发率**（是否解决了需求）  
2) **诊断完成率**（是否愿意进一步行动） citeturn2search0  
3) **演示预约/会员转化**（最终结果）  

每周只做一件事：把“高收藏但低私信”的内容改成“带模板+强CTA”的版本；把“高私信但低诊断”的内容补证据。

## 7. 至少5轮自我辩论（我把Skill压到最优的过程）
> 你要的是“最好用”，不是“最复杂”。下面是我用来砍掉无效步骤的辩论结论。

### 第1轮：要不要把步骤做得更细？
- 反方：越细越能控质量  
- 正方：越细越难执行、越难规模化  
- 结论：保留 **7步主流程**，把“模板库”做成可复用块，既控质量又能跑量。

### 第2轮：没有客户案例还能不能转化？
- 反方：没有案例没人信  
- 正方：B端最吃“流程与证据”，你站内的诊断样本/脚本/日历/质检就是证据 citeturn2search0  
- 结论：用 **Demo+样本片段+流程图** 替代“虚构案例”。虚构必翻车。

### 第3轮：SEO卡位 vs 直接投放，哪个先？
- 反方：先做SEO慢  
- 正方：纯投放会把你烧死；SEO是长期底盘  
- 结论：**70% SEO卡位 + 30% 放大验证内容（薯条/聚光）**。先用SEO打底，再用投放提速。 citeturn0search1turn0search12

### 第4轮：矩阵账号是不是越多越好？
- 反方：越多越占位  
- 正方：同质化会触发平台对低质矩阵的打击风险 citeturn1search10turn1search12  
- 结论：3个账号够了：**1个收口转化 + 2个供血信任**。更多账号先别碰。

### 第5轮：CTA到底怎么写才不死？
- 反方：直接留微信转化最高  
- 正方：平台对交易导流有明确管理，强调官方留资组件等方式，硬导流风险极高 citeturn1search0turn1search7turn1search9  
- 结论：CTA统一为 **评论关键词/私信关键词/官方留资**，站外只在合规链路后再转。
