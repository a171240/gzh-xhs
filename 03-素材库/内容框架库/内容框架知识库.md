# 公众号内容框架知识库

## 一、内容类型框架库

### 框架A：深度干货（2500-3000字）

**适用选题**：AI写作方法、批量生产SOP、系统化方法论、效率提升技巧

**结构模板**：
```
📍 痛点开场（100字）
   └─ 一句话戳中读者痛点，引发共鸣
   └─ 示例：「90%的人用AI写内容，都卡在同一个地方。」

📍 问题分析（300字）
   └─ 为什么会这样？挖掘根本原因
   └─ 用数据或案例说明问题严重性
   └─ 示例：「我观察了200+做内容的朋友，发现一个扎心的真相...」

📍 方法论展开（1500字）
   └─ 3-5个具体要点
   └─ 每个要点配真实案例
   └─ 使用「第一步」「第二步」或「要点一」「要点二」
   └─ 每个要点结构：观点+案例+数据+行动

📍 案例佐证（500字）
   └─ 我的实践经历 / 客户真实案例
   └─ 具体数据：从X到Y的变化
   └─ 示例：「拿我自己来说，以前写一条脚本要4小时...」

📍 行动建议（200字）
   └─ 读完这篇，你可以立即做什么
   └─ 给出1-3个可执行的下一步

📍 CTA引导（100字）
   └─ 回复关键词领取工具/做诊断
```

**标题公式库**：
- 数字+痛点：「90%的人不知道，AI写作最大的坑是...」
- 对比+结果：「从4小时到10分钟：我用AI写脚本的完整SOP」
- 问题+答案：「公众号没流量？可能是你犯了这3个错」
- 身份+方法：「三次创业踩坑90万后，我总结的5个教训」
- 反常识：「AI写作最大的误区：以为越智能越好」

**匹配CTA**：工具体验 / IP健康诊断

## 七、AI痕迹规避词库

### A级禁用词（必须删除）

| 禁用表达 | 问题 | 替代方案 |
|---------|------|---------|
| 「这不是X，是Y」 | 假装深刻句式 | 直接说观点 |
| 「更扎心的是...」 | 套路化转折 | 用具体事实 |
| 「你知道吗？」 | 假装互动 | 直接陈述 |
| 「总的来说...」 | 总结癖 | 省略或具体化 |
| 「我当时就懵了」 | 标准化反应 | 描述具体行为 |
| 「说白了就是...」 | 解释癖 | 直接说结论 |
| 「不仅...而且...」 | 教科书句式 | 拆成两句 |
| 「毫不夸张地说...」 | 夸张前缀 | 删除，直接说 |
| 「首先...其次...最后」 | 八股文结构 | 自然过渡 |

| AI表达 | 自然替代 |
|-------|---------|
| 很生气 | 摔门就走 |

| 异样的眼光 | 看了我三秒 |
| 我想逃走 | 转身就走 |
| 非常感动 | 眼眶一热 |
| 他很紧张 | 手心全是汗 |

## 八、排版规范

### 标题规范
- **长度**：15-25字最佳
- **数字**：必须有数字（90%、3步、5个、10倍）
- **痛点**：直击目标读者痛点
- **悬念**：适当留白引发好奇

### 段落规范
- 每段不超过4行
- 重要观点单独成段
- 用空行制造呼吸感
- 短句+长句交替使用

### 排版元素
```
【小标题】使用emoji增加视觉层次
💡 第一步：明确你的定位
📊 数据告诉你真相
⚠️ 这个坑一定要避开

【重点标注】**加粗**突出关键信息

【列表格式】
```
